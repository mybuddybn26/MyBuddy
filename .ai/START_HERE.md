## Quick Start

```
1. Read AGENTS.md      ← Master workflow
       ↓
2. Read BUDDY.md       ← Project conventions
       ↓
3. Determine task type → Load matching skills from .ai/skills/
       ↓
4. Read relevant .ai/ specs for the domain:
   ├── PROJECT.md      ← What Buddy is
   ├── ARCHITECTURE.md ← How code is structured
   ├── DESIGN.md       ← Visual rules
   ├── API.md          ← Endpoint contracts
   ├── DATABASE.md     ← Schema + queries
   ├── VOICE.md        ← Voice pipeline
   ├── PROMPTS.md      ← AI personality
   ├── SECURITY.md     ← Security rules
   ├── PERFORMANCE.md  ← Optimization rules
   ├── TESTING.md      ← How to verify
   └── DEPLOYMENT.md   ← How to ship
       ↓
5. Before architectural changes: read DECISIONS.md
       ↓
6. Before implementing: read LESSONS.md
       ↓
7. After implementing: update CHANGELOG.md
       ↓
8. Before claiming done: verify with TESTING.md
```
